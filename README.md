# GoCareer — Marketing Landing Site

Public marketing website for **GoCareer**, an AI-powered career platform that connects talent with employers. This repo is a static React single-page app (landing page + privacy policy).

**Recruiter app (separate product):** [https://gocareerrecruiter.kods.app/](https://gocareerrecruiter.kods.app/)

---

## Tech stack

| Layer | Tools |
|--------|--------|
| UI | React 19, Tailwind CSS 4 |
| Build | Vite 7 |
| Components | Radix UI + shadcn-style primitives (`src/components/ui`) |
| Motion | Motion (Framer Motion successor) |
| Charts | Recharts (showcase section) |

- **Node.js** 18.18+ (20 LTS recommended)

## Quick start

### 1. Clone and enter the project

```bash
git clone <your-repo-url>
cd go-carrier-hub-main
```

Use the folder name that matches your checkout if it differs.

### 2. Install dependencies

```bash
npm install
```

### 3. Run the development server

```bash
npm run dev
```

Open **http://localhost:8080** in your browser.

The dev server binds to all interfaces (`host: true`), so you can also open the site from another device on your LAN using your machine’s IP and port `8080`.

### 4. Production build (optional)

```bash
npm run build
npm run preview
```

`preview` serves the `dist/` folder locally (default Vite preview port, often `4173`).

## Deployment

1. Build static assets:

   ```bash
   npm run build
   ```

2. Deploy the **`dist/`** directory to any static host, for example:
   - Vercel, Netlify, Cloudflare Pages
   - Nginx / Apache serving `dist` as the web root
   - S3 + CloudFront (or similar)

3. Configure the host for **SPA-style fallback** if you add client-side routes later. For the current site, only `/` and `/?page=privacy-policy` matter; most hosts serve `index.html` for unknown paths by default.

No environment variables are required for this project.

## Customization

| What to change | Where |
|----------------|--------|
| Site title & SEO meta | `index.html` |

| Nav links | `src/components/landing/Navbar.jsx` |
| Colors / theme | `src/styles.css` |

Path alias `@/` points to `src/` (see `vite.config.js`).
